### Database Size Optimization

#### Profile Storage Efficiency

```python

{
    'id': profile_id,           # ~8 bytes
    'username': username,        # ~20 bytes average
```

**Storage Calculation:**
- 500K profiles × 82 bytes = **41 MB** (global_usernames table)
- Raw scraped profiles: **~100 MB** (includes duplicates)
- Assignments + campaigns: **~50 MB**
- **Total: ~200 MB** (well under 500 MB limit)

<Tip>
The schema is intentionally minimal. Profile photos, bios, and other metadata are **not stored** to conserve space.
</Tip>

### Rate Limiting Protection

Automated delays prevent overwhelming free-tier infrastructure.

<Accordion title="Rate Limiting Implementation">
```python
for i in range(0, len(raw_records), batch_size):
    batch = raw_records[i:i + batch_size]
    
    # Single INSERT with multiple records
    supabase.table('raw_scraped_profiles').insert(batch).execute()
    inserted_raw += len(batch)
    
    # Rate limiting protection for Supabase Free Tier
    if i + batch_size < len(raw_records):
        time.sleep(rate_limit_delay)  # 100ms default
```
</Accordion>

**Performance Impact:**
- 500K profiles = 500 batches
- 500 batches × 100ms delay = **50 seconds** of rate limiting
- + 3 minutes processing time
- **Total: 3.5 minutes** for 500K profiles

<Note>
Despite the delays, this is still **1000x faster** than the old synchronous approach that would take hours.
</Note>

```python
# Exponential backoff prevents wasted retries
for attempt in range(max_retries):
    try:
        if attempt > 0:
            backoff_delay = 2 ** attempt  # 0s, 2s, 8s, 32s
            time.sleep(backoff_delay)
        run = client.actor(actor_key).call(run_input=run_input)
        break
    except Exception as e:
```

## Troubleshooting Free Tier Issues

### Issue: Database Size Approaching 500 MB

**Solutions:**
1. Delete old `raw_scraped_profiles` records
2. Archive completed campaigns
3. Remove unused profiles

```sql
-- Delete raw records older than 90 days
DELETE FROM raw_scraped_profiles
WHERE scraped_at < NOW() - INTERVAL '90 days';

-- Archive completed campaigns
DELETE FROM daily_assignments
WHERE campaign_id IN (
    SELECT campaign_id FROM campaigns
    WHERE created_at < NOW() - INTERVAL '60 days'
);
```

### Issue: Redis Memory Exceeded

**Solutions:**
1. Reduce result expiration time
2. Enable result compression
3. Clear old task results

```python
# Reduce expiration to 1 hour
result_expires = 3600

# Enable compression (default)
result_compression = 'gzip'
```

### Issue: Heroku Dyno Memory Quota Exceeded (R14)

**Solutions:**
1. Verify connection pooling is enabled
2. Reduce batch size
3. Add worker memory cleanup

```python
# More aggressive worker recycling
worker_max_tasks_per_child = 25  # Restart after 25 tasks instead of 50
```

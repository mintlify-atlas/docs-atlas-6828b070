## Overview

Deploy the Instagram Scraper API to Heroku with full async task processing capabilities. This guide covers setting up the web server, Celery workers, Redis queue, and all necessary configurations for production use.

<Note>
  The Procfile includes three process types: `web` (Flask API), `worker` (Celery task processor), and `beat` (scheduled tasks).
</Note>

## Prerequisites

Before deploying, ensure you have:

- Heroku account and CLI installed
- Git repository with the scraper source code
- Supabase account with database configured
- Apify API key for scraping operations
- Airtable API token (optional, for VA distribution)

## Deployment Steps

<Steps>
  <Step title="Create Heroku App">
    Initialize a new Heroku application:

    ```bash
    heroku create your-app-name
    ```

    This creates a new app and adds the Heroku remote to your git repository.
  </Step>

  <Step title="Add Redis Addon">
    Install Heroku Redis for the Celery task queue:

    <CodeGroup>
      ```bash Free Tier
      heroku addons:create heroku-redis:mini
      ```

      ```bash Production
      heroku addons:create heroku-redis:premium-0
      ```
    </CodeGroup>

    <Info>
      The `REDIS_URL` environment variable is automatically set when you add the Redis addon.
    </Info>
  </Step>

  <Step title="Configure Environment Variables">
    Set all required environment variables for production:

    ```bash
    # Flask configuration
    heroku config:set FLASK_ENV=production
    heroku config:set PORT=5001
    heroku config:set SECRET_KEY=$(openssl rand -hex 32)

    # Supabase configuration
    heroku config:set SUPABASE_URL=https://your-project.supabase.co
    heroku config:set SUPABASE_SERVICE_ROLE_KEY=your-service-role-key

    # Apify scraping configuration
    heroku config:set APIFY_API_KEY=your-apify-key
    heroku config:set INSTAGRAM_APIFY_ACTOR_ID=your-actor-id

    # Airtable configuration (optional)
    heroku config:set AIRTABLE_ACCESS_TOKEN=your-airtable-token
    heroku config:set AIRTABLE_BASE_ID=your-base-id
    heroku config:set NUM_VA_TABLES=80

    # CORS configuration
    heroku config:set ALLOWED_ORIGINS=https://your-frontend.com

    # Optional: Sentry error tracking
    heroku config:set SENTRY_DSN=your-sentry-dsn
    heroku config:set APP_VERSION=1.0.0
    ```

    <Warning>
      Never commit sensitive keys to git. Always use environment variables.
    </Warning>
  </Step>

  <Step title="Deploy Application">
    Push your code to Heroku:

    ```bash
    git push heroku main
    ```

    Heroku will automatically detect the Python runtime and install dependencies from `requirements.txt`.
  </Step>

  <Step title="Apply Database Migrations">
    Run the database indexes SQL in your Supabase dashboard:

    1. Navigate to Supabase SQL Editor
    2. Click "+ New Query"
    3. Copy contents of `database_indexes.sql`
    4. Click "Run"

    <Info>
      This step is critical for performance - it reduces query times from 30-60s to under 1s.
    </Info>
  </Step>

  <Step title="Scale Dynos">
    Configure your dyno formation based on workload:

    <CodeGroup>
      ```bash Development
      heroku ps:scale web=1:basic worker=1:basic
      ```

      ```bash Production (500K accounts)
      heroku ps:scale web=1:standard-1x worker=4:standard-1x
      ```

      ```bash High Scale (1M accounts)
      heroku ps:scale web=2:standard-1x worker=16:performance-l
      ```
    </CodeGroup>

    See [scaling.mdx](/deployment/scaling) for detailed scaling recommendations.
  </Step>

  <Step title="Verify Deployment">
    Test the deployment:

    ```bash
    # Check app health
    curl https://your-app.herokuapp.com/health

    # View logs
    heroku logs --tail

    # Check dyno status
    heroku ps
    ```

    Expected health response:
    ```json
    {
      "status": "healthy",
      "timestamp": "2026-03-14T10:00:00Z"
    }
    ```
  </Step>
</Steps>

## Procfile Configuration

The application uses the following process definitions:

```text Procfile
web: gunicorn wsgi:app --workers 2 --timeout 120 --log-file - --max-requests 1000 --max-requests-jitter 50
worker: celery -A celery_config worker --loglevel=info --concurrency=2 --max-tasks-per-child=50
beat: celery -A celery_config beat --loglevel=info
```

### Process Breakdown

<AccordionGroup>
  <Accordion title="web - Flask API Server">
    Runs the Gunicorn WSGI server with:
    - **2 workers** for handling concurrent requests
    - **120s timeout** for long-running requests
    - **1000 max requests** per worker before restart (prevents memory leaks)
    - **50 request jitter** for staggered worker restarts
  </Accordion>

  <Accordion title="worker - Celery Task Processor">
    Processes background scraping tasks with:
    - **Concurrency 2** - processes 2 tasks simultaneously
    - **Max 50 tasks per child** - restarts worker after 50 tasks (memory management)
    - **Info-level logging** for production monitoring
  </Accordion>

  <Accordion title="beat - Scheduled Tasks (Optional)">
    Runs periodic tasks like cleanup or scheduled scrapes:
    - Only needed if using scheduled tasks
    - Not required for API-triggered scraping
  </Accordion>
</AccordionGroup>

## WSGI Entry Point

The production entry point in `wsgi.py`:

```python wsgi.py
"""
WSGI entry point for production deployment.
"""
import os
from app import app

if __name__ == "__main__":
    port = int(os.environ.get("PORT", 5001))
    app.run(host='0.0.0.0', port=port)
```

## Celery Configuration

Production Celery settings from `celery_config.py`:

```python celery_config.py
import os
from celery import Celery
from kombu import Exchange, Queue

celery = Celery('instagram_scraper')
redis_url = os.getenv('REDIS_URL', 'redis://localhost:6379/0')

# SSL/TLS for Heroku Redis
if redis_url.startswith('redis://') and 'localhost' not in redis_url:
    redis_url = redis_url.replace('redis://', 'rediss://')
    import ssl
    broker_use_ssl = {'ssl_cert_reqs': ssl.CERT_NONE}

celery.conf.update(
    broker_url=redis_url,
    result_backend=redis_url,
    task_time_limit=7200,  # 2 hours
    worker_max_tasks_per_child=50,
    result_expires=7200,
    result_compression='gzip'
)
```

<Tip>
  The configuration automatically handles Heroku's SSL Redis requirement by converting `redis://` to `rediss://`.
</Tip>

## Monitoring Production

### View Logs

```bash
# All logs
heroku logs --tail

# Worker logs only
heroku logs --tail --dyno worker

# Search for errors
heroku logs | grep ERROR

# Last 500 lines
heroku logs -n 500
```

### Check Dyno Status

```bash
# View running dynos
heroku ps

# Restart specific dyno type
heroku restart worker
heroku restart web

# Restart all dynos
heroku restart
```

### Monitor Redis Queue

```bash
# Connect to Redis CLI
heroku redis:cli

# Check queue length
> LLEN celery

# View queue stats
> INFO stats
```

### Database Monitoring

Query job status directly in Supabase:

```sql
SELECT job_id, status, progress, profiles_scraped
FROM scrape_jobs
WHERE status IN ('queued', 'processing')
ORDER BY created_at DESC;
```

## Troubleshooting

### API Returns 500 Error

```bash
# Check logs for stack traces
heroku logs --tail

# Verify environment variables
heroku config

# Test Redis connection
heroku redis:cli
> PING

# Restart application
heroku restart
```

### Jobs Not Processing

```bash
# Check worker count
heroku ps | grep worker

# Scale up workers if needed
heroku ps:scale worker=2

# Check Redis queue
heroku redis:cli
> LLEN celery

# View worker logs
heroku logs --tail --dyno worker
```

### Memory Issues (R14 errors)

```bash
# Check memory usage in logs
heroku logs | grep "R14"

# Upgrade dyno type
heroku ps:scale web=1:standard-2x

# Or reduce batch size in code
# Edit batch_processor.py: BATCH_SIZE = 500
```

### Connection Pool Exhausted

<Warning>
  Supabase free tier limits connections to 50. The app uses connection pooling to stay within limits.
</Warning>

Check connection pooling in logs:
```bash
heroku logs | grep "connection pool"
```

The app uses singleton pattern to reuse connections:
```python
# Automatically handled in app.py
_supabase_client = None

def get_supabase_client():
    global _supabase_client
    if _supabase_client is None:
        _supabase_client = create_client(
            supabase_url,
            supabase_key,
            options=ClientOptions(auto_refresh_token=False)
        )
    return _supabase_client
```

## Cost Optimization

### Free Tier Limits

- **1 free web dyno** (sleeps after 30min inactivity)
- **No free worker dynos** (workers require paid dynos)
- **Mini Redis**: $3/month (required for Celery)

### Recommended Minimum

```bash
heroku ps:scale web=1:basic worker=1:basic
# Cost: ~$14/month ($7 web + $7 worker)
```

### Production Setup

```bash
heroku ps:scale web=1:standard-1x worker=4:standard-1x
heroku addons:upgrade heroku-redis:premium-0
# Cost: ~$75/month ($25 web + $100 workers + $15 Redis)
```

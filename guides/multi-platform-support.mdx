## Platform Configuration

Each platform requires a specific Apify actor ID configured via environment variables:

### Environment Variables

```bash .env
# Legacy Instagram support (backward compatible)
APIFY_API_KEY=apify_api_XXXXXXXXXXXXXXXXXXXXXXXX
APIFY_ACTOR_ID=your-instagram-actor-id

# Platform-specific actors (recommended)
INSTAGRAM_APIFY_ACTOR_ID=instagram-actor-id
TIKTOK_APIFY_ACTOR_ID=tiktok-actor-id
THREADS_APIFY_ACTOR_ID=threads-actor-id
X_APIFY_ACTOR_ID=twitter-actor-id
```

<Info>
  If `INSTAGRAM_APIFY_ACTOR_ID` is not set, the system falls back to the legacy `APIFY_ACTOR_ID` for Instagram scraping.
</Info>

### Actor ID Resolution

The `get_actor_id_for_platform` function handles platform-to-actor mapping:

```python utils/scraper.py
def get_actor_id_for_platform(platform: str = 'instagram') -> str:
    """
    Get the appropriate Apify actor ID for the specified platform.
    """
    platform = platform.lower()
    
    platform_env_map = {
        'instagram': 'INSTAGRAM_APIFY_ACTOR_ID',
        'threads': 'THREADS_APIFY_ACTOR_ID',
        'tiktok': 'TIKTOK_APIFY_ACTOR_ID',
        'x': 'X_APIFY_ACTOR_ID'
    }
    
    # Fallback to legacy APIFY_ACTOR_ID for Instagram
    if platform == 'instagram':
        actor_id = os.getenv('INSTAGRAM_APIFY_ACTOR_ID') or os.getenv('APIFY_ACTOR_ID')
    elif platform in platform_env_map:
        actor_id = os.getenv(platform_env_map[platform])
    else:
        # Unknown platform - default to Instagram actor
        logger.warning(f"Unknown platform '{platform}', defaulting to Instagram")
        actor_id = os.getenv('INSTAGRAM_APIFY_ACTOR_ID') or os.getenv('APIFY_ACTOR_ID')
    
    if not actor_id:
        raise ValueError(
            f"Apify actor ID not configured for platform '{platform}'. "
            f"Please set {platform_env_map.get(platform, 'APIFY_ACTOR_ID')} environment variable."
        )
    
    return actor_id
```

## Platform-Specific Input Formats

Each Apify actor expects different input formats. The scraper handles this automatically:

### Instagram

```python
run_input = {
    "usernames": ["nike", "adidas"],
    "max_count": 5
}
```

### TikTok

```python
# Clean usernames: remove @ prefix and URL parts
clean_usernames = []
for username in accounts:
    clean_username = username.strip()
    if clean_username.startswith('@'):
        clean_username = clean_username[1:]
    if 'tiktok.com/' in clean_username:
        clean_username = clean_username.split('tiktok.com/')[-1]
        if clean_username.startswith('@'):
            clean_username = clean_username[1:]
    clean_usernames.append(clean_username)

run_input = {
    "usernames": clean_usernames,
    "numFollowers": 5
}
```

<Info>
  TikTok accepts various username formats: `@username`, `username`, or full URLs like `https://www.tiktok.com/@username`
</Info>

### Threads

```python
run_input = {
    "usernames": ["zuck", "instagram"],
    "maxFollowers": 5
}
```

### X (Twitter)

```python
run_input = {

    "maxFollowers": 5,
    "maxFollowings": 5

```

<Warning>
  X/Twitter actor uses `user_names` (with underscore) instead of `usernames` used by other platforms.

## Gender Detection Across Platforms

Gender filtering works consistently across all platforms using the `full_name` field:

```python
# From tasks.py:148-152
# Step 2: Detect gender
followers_gender = detect_gender(followers)

# Step 3: Filter by target gender
filtered_followers = filter_by_gender(followers_gender, target_gender)
```

The gender detection algorithm:
- Uses `full_name` for analysis (normalized across platforms)
- Supports both "male" and "female" filters
- Falls back to "unknown" for ambiguous names
- Platform-agnostic (works the same for all platforms)

<Note>
  For TikTok, `full_name` maps to `nickname` (display name). For X, it maps to `name` (not `screen_name`).
</Note>

## Platform-Specific Retry Logic

The scraper includes automatic retry with exponential backoff for all platforms:

```python utils/scraper.py
# From utils/scraper.py:154-200
last_error = None
for attempt in range(max_retries):  # Default: 3 attempts
    try:
        # Calculate backoff delay (0s, 2s, 8s)
        if attempt > 0:
            backoff_delay = 2 ** attempt
            logger.info(f"Retry attempt {attempt + 1}/{max_retries} after {backoff_delay}s")
            time.sleep(backoff_delay)
        
        # Call Apify actor
        run = client.actor(actor_key).call(run_input=run_input)
        
        # Fetch results
        data = []
        for item in client.dataset(run["defaultDatasetId"]).iterate_items():
            data.append(item)
        
        break  # Success!
        
    except Exception as e:
        last_error = e
        error_msg = str(e).lower()
        
        # Handle specific error types
        if 'rate limit' in error_msg or '429' in error_msg:
            logger.warning(f"Rate limit hit (attempt {attempt + 1}/{max_retries})")
            if attempt < max_retries - 1:
                continue
        elif 'connection' in error_msg or 'timeout' in error_msg:
            logger.warning(f"Network error (attempt {attempt + 1}/{max_retries})")
            if attempt < max_retries - 1:
                continue
        
        # Last attempt failed
        if attempt == max_retries - 1:
            raise Exception(f"Failed after {max_retries} attempts: {last_error}")
```

## Troubleshooting

### Platform Not Configured

**Error:**
```json
{
  "success": false,
  "error": "Apify actor ID not configured for platform 'tiktok'. Please set TIKTOK_APIFY_ACTOR_ID environment variable."
}
```

**Solution:** Add the missing environment variable to your `.env` file and restart the server.

### Invalid Platform

**Error:**
```json
{
  "success": false,
  "error": "Invalid platform 'facebook'. Must be one of: instagram, tiktok, threads, x"
}
```

**Solution:** Use only supported platforms: `instagram`, `tiktok`, `threads`, or `x`.

### TikTok Username Cleaning Issues

**Problem:** TikTok usernames with @ or URLs not working

**Solution:** The scraper automatically cleans TikTok usernames. You can provide any format:
- `@username` → cleaned to `username`
- `https://www.tiktok.com/@username` → cleaned to `username`
- `username` → remains `username`

### X/Twitter Field Mapping

**Problem:** X/Twitter profiles have null values for follower counts

**Solution:** This is expected. The X scraper uses a simplified data model to reduce API quota usage:
```python
{
    'follower_count': 0,  # Not scraped
    'following_count': 0,  # Not scraped
    'posts_count': 0      # Not scraped
}
```

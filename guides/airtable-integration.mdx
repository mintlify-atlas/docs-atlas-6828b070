The Click Creators Scraper Server includes comprehensive Airtable integration for creating and managing daily outreach campaigns. This guide covers table creation, schema design, and multi-platform support.

## Overview

The Airtable integration automates the creation of VA (Virtual Assistant) tables for daily outreach campaigns:

- **Automated Table Creation**: Generate 80+ VA tables in a single API call
- **Standardized Schema**: Consistent field structure across all tables
- **Multi-Platform Support**: Works with Instagram, TikTok, Threads, and X data
- **Idempotent Operations**: Safe to re-run (skips existing tables)
- **Error Handling**: Graceful failure with detailed error reporting

<Info>
  A typical deployment creates 80 VA tables, each holding 180 profiles, for a total of 14,400 daily outreach targets.
</Info>

## Table Creation

### Programmatic API

Use the `create_airtable_tables` function to create tables programmatically:

```python
from utils.airtable_creator import create_airtable_tables

result = create_airtable_tables(
```

## Base ID Extraction

The system supports multiple input formats for base IDs:

### Direct Base ID

```python
base_id = 'appXYZ123ABC'
```

### Airtable URL

```python
from utils.airtable_creator import extract_base_id_from_url

url = 'https://airtable.com/appXYZ123ABC/tblTable1/viwView1'
base_id = extract_base_id_from_url(url)
# Returns: 'appXYZ123ABC'
```

### Implementation

```python utils/airtable_creator.py
def extract_base_id_from_url(url: str) -> Optional[str]:
    """
    Extract Airtable base ID from an Airtable URL.
    """
    # If it's already just a base_id (starts with 'app'), return it
    if url.startswith('app') and '/' not in url:
        return url
    
    # Extract base_id from URL
    pattern = r'https?://(?:www\.)?airtable\.com/(app[a-zA-Z0-9]+)'
    match = re.search(pattern, url)
    
    if match:
        return match.group(1)
    
    return None
```

## Base ID Validation

Validate base IDs before making API calls:

```python
from utils.airtable_creator import validate_base_id

# Valid base IDs
validate_base_id('appXYZ123ABC')  # True
validate_base_id('app1ovtHsWbF2Ae7x')  # True

# Invalid base IDs
validate_base_id('invalid')  # False
validate_base_id('tblTable123')  # False (table ID, not base ID)
validate_base_id('')  # False
```

### Implementation

```python utils/airtable_creator.py
def validate_base_id(base_id: str) -> bool:
    """
    Validate that a base_id follows Airtable's format.
    """
    if not base_id:
        return False
    
    # Airtable base IDs start with 'app' followed by alphanumeric characters
    pattern = r'^app[a-zA-Z0-9]{8,20}$'
    return bool(re.match(pattern, base_id))
```

## Rate Limiting

Airtable API has rate limits. The system includes automatic delays:

```python utils/airtable_creator.py
# From airtable_creator.py:193-194
# Rate limiting: Airtable allows ~5 requests per second
time.sleep(0.25)  # 250ms delay between table creations
```

**Calculation:**
- 80 tables × 250ms delay = 20 seconds of rate limiting
- Actual table creation time: ~30-40 seconds
- Total time for 80 tables: ~50-60 seconds

<Warning>
  Removing rate limiting may result in HTTP 429 (Too Many Requests) errors from Airtable's API.
</Warning>

## Error Handling

### Duplicate Table Handling

The system gracefully handles existing tables:

```python utils/airtable_creator.py
# From airtable_creator.py:196-209
try:
    table = base.create_table(name=table_name, fields=[...])
    created_tables.append(table)
    table_ids[table_name] = table.id
    logger.info(f"✓ Successfully created table: {table_name}")
except Exception as e:
    error_msg = str(e)
    
    # Check if error is due to duplicate table name
    if 'DUPLICATE_TABLE_NAME' in error_msg or 'already exists' in error_msg.lower():
        logger.info(f"⊙ Table {table_name} already exists, skipping...")
        skipped_tables.append(table_name)
    else:
        logger.error(f"✗ Error creating table {table_name}: {error_msg}")
        failed_tables.append({'table_name': table_name, 'error': error_msg})
```

<Info>
  Re-running table creation is safe. Existing tables are skipped without errors.
</Info>

### Getting Your Airtable Token

<Steps>
  <Step title="Go to Airtable Account">
    Navigate to https://airtable.com/account
  </Step>
  
  <Step title="Generate Personal Access Token">
    Click "Generate token" in the Personal Access Tokens section
  </Step>
  
  <Step title="Set Scopes">
    Grant the following scopes:
    - `data.records:read`
    - `data.records:write`
    - `schema.bases:read`
    - `schema.bases:write`
  </Step>
  
  <Step title="Select Bases">
    Choose which bases the token can access
  </Step>
  
  <Step title="Copy Token">
    Copy the generated token and add it to your `.env` file
  </Step>
</Steps>

<Warning>
  Never commit your Airtable token to version control. Always use environment variables.
</Warning>

## Troubleshooting

### Authentication Failed

**Error:**
```
Failed to initialize Airtable API: Authentication failed
```

**Solution:**
- Verify your Personal Access Token is correct
- Check token hasn't expired
- Ensure token has required scopes (data.records:write, schema.bases:write)

### Base Not Found

**Error:**
```
Failed to initialize Airtable API: Base appXYZ123ABC not found
```

**Solution:**
- Verify base ID is correct
- Ensure token has access to this specific base
- Check base hasn't been deleted

### Rate Limit Exceeded

**Error:**
```
Error creating table: Rate limit exceeded (HTTP 429)
```

**Solution:**
- Increase delay between table creations from 250ms to 500ms
- Reduce batch size (create fewer tables at once)
- Wait a few minutes before retrying

### Duplicate Table Name

**Behavior:**
Table creation is skipped with info log:
```
⊙ Table Daily_Outreach_Table_01 already exists, skipping...
```

**Solution:**
This is expected behavior. No action needed - the table already exists and will be used.

## Complete Example

Here's a complete example of setting up Airtable integration:

```python
import os
from utils.airtable_creator import (
    create_airtable_tables,
    validate_base_id,
    extract_base_id_from_url
)

# Configuration
AIRTABLE_TOKEN = os.getenv('AIRTABLE_TOKEN')
BASE_URL = 'https://airtable.com/appXYZ123ABC/tblMain/viwGrid'
NUM_TABLES = 80

# Extract and validate base ID
base_id = extract_base_id_from_url(BASE_URL)
if not validate_base_id(base_id):
    raise ValueError(f"Invalid base_id extracted: {base_id}")

print(f"Creating {NUM_TABLES} tables in base {base_id}...")

# Create tables
result = create_airtable_tables(
    api_key=AIRTABLE_TOKEN,
    base_id=base_id,
    num_tables=NUM_TABLES
)

# Report results
if result['success']:
    print(f"✓ Success!")
    print(f"  - Created: {result['tables_created']} tables")
    print(f"  - Skipped: {result['tables_skipped']} existing tables")
    print(f"  - Failed: {result['tables_failed']} tables")
    
    if result['failed_tables']:
        print("\nFailed tables:")
        for failure in result['failed_tables']:
            print(f"  - {failure['table_name']}: {failure['error']}")
    
    print(f"\nTotal table IDs: {len(result['table_ids'])}")
else:
    print(f"✗ Failed: {result['error']}")
```

## Overview

Click Creators Scraper Server is a production-ready Flask API that provides asynchronous task processing for scraping social media followers at scale. Built with Celery and Redis, it can handle 500K+ accounts with a 103x performance improvement over traditional synchronous approaches.

## Architecture Overview

The system uses a distributed architecture to handle high-volume scraping workloads:

```
Client → Flask API → Redis Queue → Celery Workers → Apify Scraper
                                         ↓
                                   Supabase Database
```

1. **Client** submits scraping job via REST API
2. **Flask API** validates request, creates job record, and queues tasks
3. **Redis** stores tasks in queue for worker consumption
4. **Celery Workers** process batches in parallel (50 accounts per worker)
5. **Apify** performs actual social media scraping
6. **Supabase** stores results with bulk insert optimization

## System Requirements

<Note>
This system is designed to work within **Supabase free tier** limits while processing 500K+ profiles monthly.
</Note>

**Prerequisites**:
- Python 3.9+
- Redis (local or cloud)
- Supabase account
- Apify account (for scraping)
- Heroku or Render account (for production)

**Optional**:
- Airtable account (for campaign management)
- Sentry account (for error tracking)

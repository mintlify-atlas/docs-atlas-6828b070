## Overview

The batch processor provides production-ready bulk database operations optimized for massive scale (500K+ profiles) while respecting Supabase Free Tier limits. These utilities use true bulk inserts, intelligent rate limiting, and connection pooling to maximize throughput.

<Info>
**Optimizations**:
- Bulk inserts (1,000 records/batch)
- Single bulk queries instead of N individual queries
- Rate limiting (100ms delay between batches)
- Graceful fallback for failed batches
- Connection pooling for efficiency
</Info>

## Core Functions

### batch_insert_profiles

High-performance bulk profile insertion with free tier protection.

```python utils/batch_processor.py
def batch_insert_profiles(
    supabase: Client,
    profiles: List[Dict],
    base_id: str,
    batch_size: int = 1000,
    rate_limit_delay: float = 0.1
) -> Tuple[int, int, int]:
    """
    Insert profiles into Supabase using TRUE bulk inserts for maximum performance.
    
    OPTIMIZED FOR 500K+ SCALE WITH FREE TIER PROTECTION:
    - Bulk inserts (1000 records at a time) instead of individual inserts
    - Single bulk query to check existing profiles instead of N queries
    - Rate limiting protection (100ms delay between batches)
    - Graceful fallback to individual inserts if batch fails
    - Handles duplicates gracefully with upsert logic
    """
```

#### Parameters

<Accordion title="supabase: Client (required)">
Supabase client instance from `supabase-py`.

```python
from supabase import create_client
import os

supabase = create_client(
    os.getenv('SUPABASE_URL'),
    os.getenv('SUPABASE_KEY')
)
```
</Accordion>

<Accordion title="profiles: List[Dict] (required)">
List of profile dictionaries to insert. Each profile must have:
- `id` (str): Unique profile identifier
- `username` (str): Profile username
- `full_name` (str, optional): Display name

```python
profiles = [
    {
        'id': '123456789',
        'username': 'john_doe',
        'full_name': 'John Doe'
    },
    {
        'id': '987654321',
        'username': 'jane_smith',
        'full_name': 'Jane Smith'
    }
]
```
</Accordion>

<Accordion title="base_id: str (required)">
Multi-tenant identifier for data isolation. Required for all operations.

```python
base_id = 'client_abc_2026_03_14'
```

<Warning>
**CRITICAL**: Always provide a unique `base_id` per client/campaign to prevent data mixing.
</Warning>
</Accordion>

<Accordion title="batch_size: int (default: 1000)">
Number of profiles to insert per batch. Default (1,000) is optimized for Supabase Free Tier:
- Batch payload: ~100-200 KB
- Well under 8 MB payload limit
- Prevents rate limiting

```python
# Default (recommended)
batch_insert_profiles(supabase, profiles, base_id)

# Smaller batches for slower connections
batch_insert_profiles(supabase, profiles, base_id, batch_size=500)

# Larger batches for paid tier (use with caution)
batch_insert_profiles(supabase, profiles, base_id, batch_size=2000)
```
</Accordion>

<Accordion title="rate_limit_delay: float (default: 0.1)">
Delay in seconds between batches to prevent rate limiting.
- Free Tier: 0.1s (100ms) - **Recommended**
- Paid Tier: 0.05s (50ms)
- Conservative: 0.2s (200ms)

```python
# Default (recommended for free tier)
batch_insert_profiles(supabase, profiles, base_id)

# Faster (paid tier)
batch_insert_profiles(supabase, profiles, base_id, rate_limit_delay=0.05)

# Conservative (avoid throttling)
batch_insert_profiles(supabase, profiles, base_id, rate_limit_delay=0.2)
```
</Accordion>

#### Returns

Tuple of `(inserted_raw, added_to_global, skipped_existing)`:

```python
inserted_raw, added_to_global, skipped = batch_insert_profiles(
    supabase, profiles, base_id
)

print(f"Inserted into raw_scraped_profiles: {inserted_raw}")
print(f"Added to global_usernames: {added_to_global}")
print(f"Skipped (duplicates): {skipped}")
```

**Example Output**:
```

Added to global_usernames: 48532
Skipped (duplicates): 1468
```

#### Database Tables

The function inserts into two tables:

<Accordion title="raw_scraped_profiles (Historical)">
Stores ALL scraped profiles with timestamp for historical tracking.

**Schema**:
```sql
CREATE TABLE raw_scraped_profiles (
    id TEXT PRIMARY KEY,
    username TEXT NOT NULL,
    full_name TEXT,
    scraped_at TIMESTAMPTZ NOT NULL,
    base_id TEXT NOT NULL
);
```

**Purpose**: Historical record of all scrapes (allows duplicate IDs across scrapes)
</Accordion>

<Accordion title="global_usernames (Deduplicated)">
Stores unique profiles for assignment distribution.

**Schema**:
```sql
CREATE TABLE global_usernames (
    id TEXT PRIMARY KEY,
    username TEXT NOT NULL,
    full_name TEXT,
    used BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMPTZ NOT NULL,
    base_id TEXT NOT NULL
);
```

**Purpose**: Master list of unique profiles (rejects duplicate IDs)
</Accordion>

```python
    batch_insert_profiles(
        supabase=supabase,

        base_id='nike_campaign_2026_q1'
```

- ✓ Connection pooling: Single reused connection
- ✓ Database size: 500 MB limit (monitor with 500K+ profiles)


### batch_update_assignments

Bulk update daily assignments with optimized queries.

```python utils/batch_processor.py
def batch_update_assignments(
    supabase: Client,
    updates: List[Dict],
    batch_size: int = 500
) -> int:
    """
    Update daily_assignments in batches using bulk operations.
    
    OPTIMIZED FOR 500K+ SCALE:
    - Uses PostgreSQL CASE WHEN for bulk updates (100x faster than individual)
    - Processes in chunks to avoid query size limits
    - Single query per chunk instead of N individual queries
    """
```

#### Parameters

<Accordion title="supabase: Client (required)">
Supabase client instance (same as `batch_insert_profiles`).
</Accordion>

<Accordion title="updates: List[Dict] (required)">
List of update dictionaries. Each must have:
- `assignment_id` (str): Assignment identifier to update
- Additional fields to update (e.g., `va_table_number`, `status`)

```python
updates = [
    {
        'assignment_id': 'assign_001',
        'va_table_number': 1,
        'status': 'assigned'
    },
    {
        'assignment_id': 'assign_002',
        'va_table_number': 2,
        'status': 'assigned'
    }
]
```
</Accordion>

<Accordion title="batch_size: int (default: 500)">
Number of updates per batch. Lower than inserts due to update complexity.
</Accordion>

#### Returns

Number of successfully updated records (int).

#### Usage Example

```python
from utils.batch_processor import batch_update_assignments

# Prepare updates
updates = [
    {'assignment_id': 'a1', 'va_table_number': 1, 'status': 'sent'},
    {'assignment_id': 'a2', 'va_table_number': 1, 'status': 'sent'},
    {'assignment_id': 'a3', 'va_table_number': 2, 'status': 'sent'},
    # ... thousands more
]

# Bulk update
updated_count = batch_update_assignments(
    supabase=supabase,
    updates=updates,
    batch_size=500
)

print(f"Updated {updated_count} assignments")
```

<Note>
Currently implements individual updates with batched error handling. Future versions will use PostgreSQL `CASE WHEN` for true bulk updates.
</Note>

### batch_delete_records

Bulk delete records from any table.

```python utils/batch_processor.py
def batch_delete_records(
    supabase: Client,
    table_name: str,
    record_ids: List[str],
    id_column: str = 'id',
    batch_size: int = 500
) -> int:
    """
    Delete records from a table in batches.
    """
```

#### Parameters

<Accordion title="supabase: Client (required)">
Supabase client instance.
</Accordion>

<Accordion title="table_name: str (required)">
Name of the table to delete from.

**Examples**: `'raw_scraped_profiles'`, `'global_usernames'`, `'daily_assignments'`
</Accordion>

<Accordion title="record_ids: List[str] (required)">
List of record IDs to delete.

```python
record_ids = ['id1', 'id2', 'id3', ...]
```
</Accordion>

<Accordion title="id_column: str (default: 'id')">
Name of the ID column in the table.

**Common values**: `'id'`, `'assignment_id'`, `'username'`
</Accordion>

<Accordion title="batch_size: int (default: 500)">
Number of deletes per batch.
</Accordion>

#### Returns

Number of successfully deleted records (int).

#### Usage Examples

<Accordion title="Delete Old Scrapes">

## Complete Workflow

End-to-end example: scrape → detect gender → insert to database

```python
from utils.scraper import scrape_followers
from utils.gender import detect_gender, filter_by_gender
from utils.batch_processor import batch_insert_profiles
from supabase import create_client

import logging

    level=logging.INFO,
    format='%(asctime)s - %(levelname)s - %(message)s'
)

# Initialize Supabase
supabase = create_client(
    os.getenv('SUPABASE_URL'),
    os.getenv('SUPABASE_KEY')
)

# Campaign configuration
BASE_ID = 'nike_campaign_2026_q1'
TARGET_ACCOUNTS = ['nike', 'nikewomen', 'nikerunning']
TARGET_GENDER = 'female'
MAX_FOLLOWERS_PER_ACCOUNT = 1000

print("=" * 70)
print(f"Starting campaign: {BASE_ID}")
print(f"Target accounts: {TARGET_ACCOUNTS}")
print(f"Target gender: {TARGET_GENDER}")
print("=" * 70)

# Step 1: Scrape followers
print("\n[1/4] Scraping followers...")
followers = scrape_followers(
    accounts=TARGET_ACCOUNTS,
    max_count=MAX_FOLLOWERS_PER_ACCOUNT,
    platform='instagram'
)
print(f"✓ Scraped {len(followers)} total followers")

# Step 2: Detect gender
print("\n[2/4] Detecting gender...")
gender_map = detect_gender(followers)
filtered_followers = filter_by_gender(gender_map, TARGET_GENDER)
print(f"✓ Filtered to {len(filtered_followers)} {TARGET_GENDER} followers")

# Step 3: Prepare profiles for insertion
print("\n[3/4] Preparing profiles for database...")
profiles_to_insert = [
    followers[username] 
    for username in filtered_followers.keys()
    if username in followers
]
print(f"✓ Prepared {len(profiles_to_insert)} profiles")

# Step 4: Bulk insert to database
print("\n[4/4] Inserting profiles to database...")
inserted, added, skipped = batch_insert_profiles(
    supabase=supabase,
    profiles=profiles_to_insert,
    base_id=BASE_ID,
    batch_size=1000,
    rate_limit_delay=0.1
)

print("\n" + "=" * 70)
print("CAMPAIGN COMPLETE")
print("=" * 70)
print(f"Total scraped: {len(followers)}")
print(f"Gender filtered: {len(filtered_followers)}")
print(f"Inserted to raw: {inserted}")
print(f"New global profiles: {added}")
print(f"Duplicates skipped: {skipped}")
print("=" * 70)
```

## Supabase Free Tier Best Practices

### Connection Management

```python
from supabase import create_client
import os

# GOOD: Single client instance (connection pooling)
supabase = create_client(
    os.getenv('SUPABASE_URL'),
    os.getenv('SUPABASE_KEY')
)

# Use same client for all operations
batch_insert_profiles(supabase, profiles1, 'base1')
batch_insert_profiles(supabase, profiles2, 'base2')

# BAD: Creating new clients (connection leak)
for profiles in profile_batches:
    supabase = create_client(...)  # DON'T DO THIS
    batch_insert_profiles(supabase, profiles, base_id)
```

### Rate Limiting

```python
# Free Tier: 100ms delay (recommended)
batch_insert_profiles(
    supabase, profiles, base_id,
    rate_limit_delay=0.1
)

# Monitor for throttling errors
try:
    batch_insert_profiles(supabase, profiles, base_id)
except Exception as e:
    if '429' in str(e) or 'rate limit' in str(e).lower():
        # Increase delay and retry
        time.sleep(5)
        batch_insert_profiles(
            supabase, profiles, base_id,
            rate_limit_delay=0.2  # Double the delay
        )
```

### Database Size Monitoring

```python
from supabase import create_client

supabase = create_client(
    os.getenv('SUPABASE_URL'),
    os.getenv('SUPABASE_KEY')
)

# Check total profiles
total_profiles = supabase.table('global_usernames')\
    .select('id', count='exact')\
    .execute()

count = total_profiles.count
print(f"Total profiles: {count:,}")

# Free Tier: 500 MB limit
# Estimate: ~300 bytes per profile
# Max profiles: ~1.6M (but stay under 1M for safety)
if count > 1_000_000:
    print("WARNING: Approaching database size limit!")
```

## Error Handling

### Batch Failure Fallback

The batch processor automatically falls back to individual inserts if a batch fails:

```python
try:
    # Try bulk insert
    supabase.table('raw_scraped_profiles').insert(batch).execute()
    inserted_raw += len(batch)
except Exception as e:
    logger.error(f"Batch insert failed: {e}")
    # Fallback: Individual inserts
    for record in batch:
        try:
            supabase.table('raw_scraped_profiles').insert(record).execute()
            inserted_raw += 1
            time.sleep(0.01)  # 10ms delay for individual inserts
        except Exception as e2:
            logger.warning(f"Failed to insert {record['username']}: {e2}")
```

### Duplicate Handling

Duplicates are handled gracefully:

```python
# Duplicates in global_usernames are skipped (unique constraint)
try:
    supabase.table('global_usernames').insert(record).execute()
    added_to_global += 1
except Exception as e:
    if 'unique constraint' in str(e).lower():
        skipped_existing += 1  # Expected duplicate
    else:
        logger.error(f"Unexpected error: {e}")  # Real error
```

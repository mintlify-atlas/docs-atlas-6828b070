## Welcome to Click Creators Scraper Server

This asynchronous task processing system is capable of handling **1,000,000+ social media accounts per month** with a production-ready architecture built on Flask, Celery, and Redis.

<Note>
  This guide will help you go from zero to your first successful API call in under 5 minutes.
</Note>

## Prerequisites

Before you begin, make sure you have:

- Python 3.9 or higher
- Redis (local or cloud)
- Supabase account
- Apify account with API key
- Airtable account (optional)

## Installation

<Steps>

<Step title="Clone and Install Dependencies">

First, navigate to the server directory and install the required Python packages:

```bash
cd server
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

Key dependencies include:
- `flask==3.0.0` - Web framework
- `celery==5.3.4` - Background task queue
- `redis==4.6.0` - Message broker
- `supabase==2.9.0` - Database client
- `apify-client==1.7.0` - Social media scraping

</Step>

<Step title="Install and Start Redis">

Redis is required for Celery's task queue. Install and start it based on your OS:

<CodeGroup>

```bash macOS
brew install redis
brew services start redis
```

```bash Ubuntu/Debian
sudo apt-get install redis-server
sudo systemctl start redis
```

```bash Windows (WSL)
sudo apt-get install redis-server
sudo service redis-server start
```

</CodeGroup>

Verify Redis is running:

```bash
redis-cli ping
# Should return: PONG
```

</Step>

<Step title="Configure Environment Variables">

Create a `.env` file in the server directory with your credentials:

```bash .env
FLASK_ENV=development
PORT=5001

# Redis configuration
REDIS_URL=redis://localhost:6379/0

# Supabase credentials
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key

# Apify configuration
APIFY_API_KEY=your-apify-api-key
INSTAGRAM_APIFY_ACTOR_ID=your-actor-id

# Airtable (optional)
AIRTABLE_ACCESS_TOKEN=your-airtable-token

# VA Tables configuration
NUM_VA_TABLES=80
```

<Warning>
  Keep your `.env` file secure and never commit it to version control. Add it to `.gitignore`.
</Warning>

</Step>

<Step title="Apply Database Indexes">

For optimal performance, run the database indexes script once in your Supabase SQL Editor:

1. Go to Supabase Dashboard → SQL Editor
2. Click "+ New Query"
3. Copy contents of `database_indexes.sql`
4. Click "Run"

This adds 9 critical indexes that reduce query times from 30-60s to less than 1s.

</Step>

<Step title="Start the Flask API">

In your first terminal window, start the Flask application:

```bash
python app.py
```

You should see:

```
 * Running on http://localhost:5001
✓ Supabase client initialized with connection pool
✓ RLS context setup enabled for multi-tenant isolation
```

</Step>

<Step title="Start the Celery Worker">

In a **new terminal window**, start the Celery worker:

```bash
source venv/bin/activate
celery -A celery_config worker --loglevel=info --concurrency=2
```

You should see:

```
[tasks]
  . tasks.scrape_account_batch
  . tasks.aggregate_scrape_results
  . tasks.ingest_profiles_batch
  . tasks.daily_pipeline_orchestrator

celery@hostname ready.
```

</Step>

</Steps>

## Your First API Call

Now that everything is running, let's make your first scraping request!

### Health Check

First, verify the server is running:

```bash
curl http://localhost:5001/health
```

Expected response:
```json
{
  "status": "ok",
  "service": "Instagram Scraper API",
  "version": "1.0.0"
}
```

### Scrape Instagram Followers

Queue a job to scrape followers from Instagram accounts:

```bash
curl -X POST http://localhost:5001/api/scrape-followers \
  -H "Content-Type: application/json" \
  -d '{
    "accounts": ["nike", "adidas"],
    "targetGender": "male",
    "totalScrapeCount": 50
  }'
```

<Info>
  The API returns immediately (< 200ms) with a job ID. The actual scraping happens asynchronously in the background.
</Info>

Expected response:

```json
{
  "success": true,
  "job_id": "abc-123-def-456",
  "base_id": "default_instagram",
  "platform": "instagram",
  "status_url": "/api/job-status/abc-123-def-456",
  "results_url": "/api/job-results/abc-123-def-456",
  "total_batches": 1,
  "message": "Instagram scraping job queued successfully. Poll status_url for progress."
}
```

### Check Job Status

Poll the status endpoint to track progress:

```bash
curl http://localhost:5001/api/job-status/abc-123-def-456
```

Response while processing:

```json
{
  "success": true,
  "job_id": "abc-123-def-456",
  "status": "processing",
  "progress": 45.5,
  "profiles_scraped": 25,
  "total_batches": 1,
  "current_batch": 1
}
```

Response when complete:

```json
{
  "success": true,
  "job_id": "abc-123-def-456",
  "status": "completed",
  "progress": 100.0,
  "profiles_scraped": 50,
  "total_scraped": 75,
  "total_filtered": 50
}
```

### Get Results

Once the job is complete, retrieve the scraped profiles:

```bash
curl "http://localhost:5001/api/job-results/abc-123-def-456?page=1&limit=100"
```

Response:

```json
{
  "success": true,
  "job_id": "abc-123-def-456",
  "page": 1,
  "limit": 100,
  "total": 50,
  "profiles": [
    {
      "id": "123456789",
      "username": "john_athlete",
      "full_name": "John Smith",
      "created_at": "2025-10-14T12:34:56.789Z"
    }
  ]
}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Redis connection refused">
    Make sure Redis is running:
    
    ```bash
    redis-cli ping
    ```
    
    If it returns `PONG`, Redis is working. If not, restart Redis:
    
    ```bash
    # macOS
    brew services restart redis
    
    # Linux
    sudo systemctl restart redis
    ```
  </Accordion>

  <Accordion title="Celery tasks not processing">
    Check that the Celery worker is running:
    
    ```bash
    celery -A celery_config inspect active
    ```
    
    View worker logs for errors:
    
    ```bash
    celery -A celery_config worker --loglevel=debug
    ```
  </Accordion>

  <Accordion title="Job stuck in 'processing' status">
    This usually indicates the Celery worker crashed. Check worker logs:
    
    ```bash
    # View recent Celery logs
    tail -f celery.log
    ```
    
    Restart the worker:
    
    ```bash
    celery -A celery_config worker --loglevel=info --concurrency=2
    ```
  </Accordion>

  <Accordion title="Supabase connection errors">
    Verify your Supabase credentials in `.env`:
    
    ```bash
    echo $SUPABASE_URL
    echo $SUPABASE_SERVICE_ROLE_KEY
    ```
    
    Make sure you're using the **service role key**, not the anon key.
  </Accordion>
</AccordionGroup>

## Prerequisites

Before starting, ensure you have the following installed:

- **Python 3.9+** (recommended: Python 3.11)
- **Redis** (for Celery task queue)
- **Git** (for version control)
- **Virtual environment** (venv or virtualenv)

<Note>
The scraper server is built with Flask for the API layer and Celery for asynchronous task processing. Redis is required as the message broker for Celery.
</Note>

## Quick Start

The project includes a quick start script that automates most of the setup:

<Steps>
  <Step title="Clone and navigate to the project">
    ```bash
    cd server
    ```
  </Step>

  <Step title="Run the quick start script">
    ```bash
    chmod +x quickstart.sh
    ./quickstart.sh
    ```
    
    This script will:
    - Check Python version
    - Install and start Redis
    - Create a virtual environment
    - Install dependencies
    - Generate a `.env` template
    - Create test scripts
  </Step>

  <Step title="Configure environment variables">
    Update the generated `.env` file with your actual credentials (see [Environment Variables](/setup/environment-variables) for details).
  </Step>

  <Step title="Apply database migrations">
    See [Database Setup](/setup/database-setup) for instructions on creating the required tables and indexes.
  </Step>
</Steps>

## Manual Setup

If you prefer to set up manually or the quick start script doesn't work on your system:

### 1. Install Redis

<CodeGroup>
```bash macOS
# Using Homebrew
brew install redis
brew services start redis

# Verify Redis is running
redis-cli ping
# Expected output: PONG
```

```bash Ubuntu/Debian
# Install Redis
sudo apt-get update
sudo apt-get install redis-server

# Start Redis
sudo systemctl start redis
sudo systemctl enable redis

# Verify Redis is running
redis-cli ping
# Expected output: PONG
```

```bash Windows
# Using WSL2 (recommended)
wsl --install
wsl
sudo apt-get update
sudo apt-get install redis-server
sudo service redis-server start

# Or download Redis for Windows from:
# https://redis.io/download
```
</CodeGroup>

### 2. Create Virtual Environment

```bash
# Navigate to server directory
cd server

# Create virtual environment
python3 -m venv venv

# Activate virtual environment
# On macOS/Linux:
source venv/bin/activate

# On Windows:
venv\Scripts\activate
```

### 3. Install Dependencies

```bash
# Install all required packages
pip install -r requirements.txt
```

<Info>
The `requirements.txt` includes all production dependencies:
- Flask 3.0.0 (web framework)
- Celery 5.3.4 (task queue)
- Supabase 2.9.0 (database client)
- Apify Client 1.7.0 (scraping)
- Gunicorn 21.2.0 (production server)
- And more...
</Info>

### 4. Configure Environment Variables

Create a `.env` file in the `server/` directory:

```bash
touch .env
```

See the [Environment Variables](/setup/environment-variables) page for the complete configuration.

### 5. Setup Database

Follow the [Database Setup](/setup/database-setup) guide to create the required Supabase tables and indexes.

## Running the Development Server

You need to run **three services** simultaneously:

### Option A: Using tmux (Recommended)

The quick start script generates a `start_all.sh` script that uses tmux:

```bash
./start_all.sh
```

This will start all services in a tmux session with split panes:
- **Pane 0**: Flask API server
- **Pane 1**: Celery worker
- **Pane 2**: Test terminal

<Note>
To exit tmux: Press `Ctrl+B` then `D` to detach. To reattach: `tmux attach -t instagram_api`
</Note>

### Option B: Manual in Separate Terminals

<Steps>
  <Step title="Terminal 1: Start Flask API">
    ```bash
    source venv/bin/activate
    python app.py
    ```
    
    The API will start on `http://localhost:5001`
    
    **Expected output:**
    ```
    [INFO] Flask app initialized with CORS and rate limiting
    [INFO] ✓ Supabase client initialized with connection pool (size: 5, tier: free)
    [INFO] ✓ RLS context setup enabled for multi-tenant isolation
     * Running on http://127.0.0.1:5001
    ```
  </Step>

  <Step title="Terminal 2: Start Celery Worker">
    ```bash
    source venv/bin/activate
    celery -A celery_config worker --loglevel=info --concurrency=2
    ```
    
    **Expected output:**
    ```

    -- * - **** ---
    ```

  <Step title="Terminal 3: Test the API">

    Or test manually:

## Development Workflow

### Making Code Changes

<Steps>
  <Step title="Edit code">
    Make your changes to any `.py` files.
  </Step>

  <Step title="Restart services">
    - **Flask**: Press `Ctrl+C` and run `python app.py` again
    - **Celery**: Press `Ctrl+C` and run the celery command again
    
    <Warning>
    Celery does **not** auto-reload on code changes. You must restart the worker after modifying task code.
    </Warning>
  </Step>

  <Step title="Test changes">
    Use the test scripts or make API calls to verify your changes.
  </Step>
</Steps>

### Hot Reloading

For Flask auto-reload during development:

```python
# In app.py (last lines)
if __name__ == '__main__':
    debug = os.getenv('FLASK_ENV') != 'production'
    port = int(os.getenv('PORT', 5001))
    app.run(host='0.0.0.0', port=port, debug=debug)  # debug=True enables hot reload
```

## Troubleshooting

### Redis Connection Failed

**Error:**
```
celery.exceptions.OperationalError: Error connecting to Redis
```

**Solution:**
```bash
# Check if Redis is running
redis-cli ping

# If not, start Redis
# macOS:
brew services start redis

# Linux:
sudo systemctl start redis
```

### Port Already in Use

**Error:**
```
OSError: [Errno 48] Address already in use
```

**Solution:**
```bash
# Find process using port 5001
lsof -i :5001

# Kill the process
kill -9 <PID>

# Or use a different port
export PORT=5002
python app.py
```

### Celery Tasks Not Running

**Symptoms:** Jobs stay in "queued" status

**Solution:**
1. Check Celery worker is running:
   ```bash
   celery -A celery_config inspect active
   ```

2. Check Redis queue:
   ```bash
   redis-cli
   > LLEN celery
   ```

3. Restart Celery worker:
   ```bash
   # Ctrl+C to stop, then restart
   celery -A celery_config worker --loglevel=info --concurrency=2
   ```

### Import Errors

**Error:**
```
ModuleNotFoundError: No module named 'flask'
```

**Solution:**
```bash
# Ensure virtual environment is activated
source venv/bin/activate

# Reinstall dependencies
pip install -r requirements.txt
```

### Database Connection Issues

**Error:**
```
ValueError: SUPABASE_URL and SUPABASE_SERVICE_ROLE_KEY environment variables are required.
```

**Solution:**
1. Check `.env` file exists and has correct values
2. Restart Flask app to load new environment variables
3. Verify Supabase credentials are correct

## Performance Tips

<Tip>
**Local Development Optimizations:**
- Use `--concurrency=2` for Celery (matches development workload)
- Enable Flask debug mode for hot reloading
- Use Redis locally (don't use cloud Redis for dev)
- Keep `SUPABASE_POOL_SIZE=5` for local testing
</Tip>

<Warning>
**Do Not:**
- Run multiple Celery workers on the same machine (causes race conditions)
- Use production credentials in `.env` for local development
- Commit `.env` to version control
- Test with large scraping jobs (>1000 profiles) locally
</Warning>
